# Run Assistant

`conversation_express_system` を日常的に起動するためのスキル。セットアップ済みの環境を前提に、起動に必要な一連の流れを自動で実行する。

## トリガー

ユーザーが `run` と打ったとき。

---

## 基本方針

- **状態をまず観測する。** 各ステップで現状をチェックしてから、必要なときだけ実行する。
- **仮想環境の activate は行わない。** `source .venv/bin/activate` は子プロセスの環境変数が親シェルに引き継がれないため機能しない。`.venv/bin/pip` / `.venv/bin/python` を直接呼ぶ。
- **実行前に承認は不要。** このスキルは日常起動用なので、確認なしで順に実行する。ただし想定外のエラーが出たら止まって報告する。

---

## ステップ 1: VOICEVOX 起動確認

`config.py` を Read して `VOICEVOX_URL` を取得し、以下で確認する：

```bash
curl -sf "${VOICEVOX_URL}/version"
```

- 成功（バージョン文字列が返る）→ 「VOICEVOX は起動中」と表示して次へ
- 失敗 → 以下を案内して**終了**する：

> VOICEVOX が起動していません。
> VOICEVOX アプリを起動してエンジンが立ち上がったら、もう一度 `run` と打ってください。

（VOICEVOX は GUI アプリのため自動起動できない）

---

## ステップ 2: Ollama サーバ起動確認

```bash
curl -sf http://127.0.0.1:11434/api/tags
```

- 成功（JSON が返る）→ 「Ollama はすでに起動中」と表示し次へ
- 失敗 → **自分で起動せず、ユーザーに指示を出して終了する**：

> Ollama が起動していません。別ターミナルで以下を実行してください：
>
> ```
> ollama serve
> ```
>
> 起動したら、もう一度 `run` と打ってください。

（このスキルから `ollama serve` を実行してはいけない）

---

## ステップ 3: 仮想環境の確認

```bash
test -d .venv && echo "exists"
```

- `exists` が返る → 次へ
- 返らない → 「`.venv` が見つかりません。先に `setup` を実行してください」と案内して終了

---

## ステップ 4: 依存パッケージのインストール

```bash
.venv/bin/pip install -r requirements.txt -q
```

完了したら「依存パッケージのインストール完了」と表示して次へ。

---

## ステップ 5: app.py の起動指示

**自分では起動せず、ユーザーに指示を出して終了する**：

> 準備が整いました。以下を別ターミナルで実行して app を起動してください：
>
> ```
> .venv/bin/python app.py
> ```
>
> 起動したら、ブラウザで http://127.0.0.1:5000 にアクセスしてください。

（このスキルから `python app.py` を実行してはいけない）

---

## エラー時の対処方針

- `pip install` が失敗 → エラーログを表示し、原因（portaudio 等のシステムライブラリ不足が多い）を切り分けて案内
- ユーザーが `ollama serve` / `app.py` を起動して問題が起きた場合は、ターミナル出力のエラー行を貼ってもらい切り分けを行う
