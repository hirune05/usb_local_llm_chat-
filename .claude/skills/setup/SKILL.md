# Setup Assistant

このリポジトリ（USBポータブル ローカルLLM会話システム）を**開発モード**で初めて動かす人を、対話形式でセットアップ完了まで導くスキル。

> **注意**: これは開発環境（`python app.py` で動かす）用のセットアップ。
> USB版を使うだけなら環境構築は不要（USB内の `start_mac.command` をダブルクリック）。
> USB一式をゼロから作る手順は `BUILD.md` を参照。

## トリガー

ユーザーが `setup` と打ったとき。

---

## 基本方針

- **状態をまず観測する。** 各ステップで現状をチェックしてから、必要なときだけ案内・実行する。
- **実行前に必ず承認を取る。** `brew install`, `ollama pull`, `pip install` など副作用のあるコマンドは、ユーザーが承認してから実行する。
- **GUI アプリ（VOICEVOX）は起動できない。** 案内のみ行い、ユーザーが起動したら再チェックする。
- **値は config.py から動的に取得する。** モデル名・URL を Skill 内にハードコードしない。
- **対象環境は macOS を前提**（`brew`, `open` コマンドを使うため）。Linux/Windows ユーザーには適宜読み替えを案内。

---

## ステップ 0: 設定値を読み取る

最初に `config.py` を Read して、以下の値を取得する：
- `LLM_MODEL`（例: `gemma4:e2b-it-qat`）
- `VOICEVOX_URL`（例: `http://127.0.0.1:50021`）

以降の URL・モデル名はこれを使う。**ハードコードしないこと。**

ユーザーに「セットアップを始めます。順にチェックしますね」と一言伝える。

---

## ステップ 1: Python バージョン確認

```bash
python3 --version
```

- 3.10 以上 → OK、次へ
- 3.10 未満 → 「`faster-whisper` などが Python 3.10+ を推奨します。pyenv 等でアップデートを検討してください」と案内したうえで、続行するか確認する

---

## ステップ 2: VOICEVOX 起動確認

```bash
curl -sf "${VOICEVOX_URL}/version"
```

- 成功（バージョン文字列が返る）→ OK、次へ
- 失敗 → 以下を案内：
  > VOICEVOX が起動していません。
  > 1. インストール済みでない場合: https://voicevox.hiroshiba.jp/ からダウンロード
  > 2. アプリを起動（エンジンが自動で立ち上がります）
  > 3. 起動できたら「OK」と返信してください

ユーザーが「OK」など返したら再チェック。

---

## ステップ 3: Ollama インストール確認

```bash
command -v ollama
```

- 見つかる → OK、次へ
- 見つからない → `brew install ollama` を提案。承認を取ってから実行

---

## ステップ 4: Ollama サーバ起動確認

```bash
curl -sf http://127.0.0.1:11434/api/tags
```

- 成功（JSON が返る）→ OK、次へ
- 失敗 → 以下を案内：
  > Ollama サーバが起動していません。**別ターミナル**を開いて以下を実行してください：
  > ```bash
  > ollama serve
  > ```
  > （このターミナルは開いたままにしてください）
  > 起動できたら「OK」と返信してください

ユーザーが返答したら再チェック。

---

## ステップ 5: モデル取得確認

ステップ 0 で取得した `LLM_MODEL` が `ollama list` に含まれるか確認：

```bash
curl -s http://127.0.0.1:11434/api/tags | grep -q "${LLM_MODEL}"
```

- 含まれる → OK、次へ
- 含まれない → 「`ollama pull ${LLM_MODEL}` を実行します。**数 GB のダウンロードがあるので時間がかかります**」と案内し、承認後に実行

---

## ステップ 6: Python 仮想環境

```bash
test -d .venv
```

- 存在する → スキップ
- 存在しない → `python3 -m venv .venv` を承認後に実行

**activate について：**
> 仮想環境の `source .venv/bin/activate` は、このセッションでは自動実行できません（子プロセスの環境変数は親シェルに引き継がれないため）。
> 別ターミナルで作業する際は手動で `source .venv/bin/activate` してください。
> このスキルでは `pip` と `python` を `.venv/bin/` から直接呼ぶので、続行はそのまま可能です。

---

## ステップ 7: 依存パッケージ導入

```bash
.venv/bin/pip install -r requirements.txt
```

承認後に実行。完了後 `.venv/bin/pip list | head -20` などで主要パッケージが入ったことを軽く確認すると親切。

---

## ステップ 8: マイク権限の案内

> このアプリは音声録音に `sounddevice` を使います。
> 初回起動時に macOS のマイクアクセス許可ダイアログが出ます。
> もし出ない・拒否してしまった場合は、**システム設定 → プライバシーとセキュリティ → マイク** で、ターミナル（または iTerm / VS Code など使用中のアプリ）にチェックを入れてください。

---

## ステップ 9: 起動 & ブラウザ確認

最終確認：「ここまで全て準備完了です。アプリを起動しますか？」

承認後：

```bash
.venv/bin/python app.py
```

をバックグラウンドで起動し、出力に `サーバを http://127.0.0.1:5001 で起動します` が出るまで待つ
（ポート 5000 は macOS の AirPlay Receiver と衝突するため、このアプリの既定は **5001**。`APP_PORT` 環境変数で変更可）。

その後ユーザーに以下を案内：

> 起動しました。ブラウザで開きます：
> ```bash
> open http://127.0.0.1:5001
> ```
> もしくは手動で http://127.0.0.1:5001 にアクセスしてください。

`open` コマンドの実行も承認後に行う。

---

## 完了後のメッセージ

- 動作確認のポイント（チャット欄に文字を打つと音声＋表情が返る）を一言伝える
- もしエラーが出たら、ターミナルの `python app.py` のログを共有してもらえれば原因を一緒に追える旨を伝える

---

## エラー時の対処方針

- VOICEVOX/Ollama サーバの応答待ちで `curl` が失敗したら、3 回までリトライ案内する
- `pip install` が失敗したら、エラーメッセージを読み取って原因（macOS の場合 portaudio 等のシステムライブラリ不足が多い）を切り分ける
- どうしても解決しない場合は、エラーログを丸ごと共有してもらい、ステップ単位で再実行できるようにする
